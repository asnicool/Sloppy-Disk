# Artist Images Feature - Implementation Plan

## Goal
Automatically fetch and display artist photos from the same web sources used for album cover art (Discogs, MusicBrainz/Cover Art Archive). Cache locally in `coverArtRoot/{artist-name}/Artist.jpg`. Display automatically without user interaction, with management option only in the artist page.

---

## 1. Architecture Overview

### Storage Location
- **Location**: `{CoverArtRoot}/{artist-name}/Artist.jpg`
- **Fallback**: If `CoverArtRoot` not set, use `{MusicRoot}/{artist-name}/Artist.jpg`
- **File naming**: `Artist.jpg` (with fallback to `Artist.png`, `Artist.webp`)

### Display Flow (Automatic)
1. **Search Results - Artists Section**: Show artist image thumbnail (auto-displayed)
2. **Artist Page**: Show large artist image (auto-displayed)
3. **No manual selection needed** - First match is used automatically

### Management Flow (Manual - only when wrong)
- **Artist Page only**: "Update" button to fetch new candidates and select alternative
- **No management in Search results**

### Web Sources (reuse existing providers)
1. **Discogs** - Has artist images via `/artists/{mbid}` endpoint (best source)
2. **MusicBrainz/Cover Art Archive** - Via artist-credits URL relations (fanart.tv, wikipedia)
3. **AlbumArt.digital** - Not suitable (album-focused)

## 2. Backend Implementation

### 2.1 New Config Options
```go
// config/config.go
// No new config needed - reuse existing CoverArtRoot
```

### 2.2 New Provider Methods
Add to existing `metadata` providers:

**Discogs** - Use artist search endpoint (`/database/search?type=artist`) then `/artists/{mbid}` to get images:
```go
// GetArtistImage fetches artist photo from Discogs
func (p *DiscogsProvider) GetArtistImage(artistName string) ([]models.ArtistImageCandidate, error)
```

**MusicBrainz** - Use artist lookup with "url-rels" to find fanart.tv, wikidata, etc.

### 2.3 New Models
```go
// models/models.go

// ArtistImageCandidate represents an artist image from external sources
type ArtistImageCandidate struct {
    Source    string `json:"source"`
    URL       string `json:"url"`
    Thumbnail string `json:"thumbnail,omitempty"`
    Width     int    `json:"width,omitempty"`
    Height    int    `json:"height,omitempty"`
    License   string `json:"license,omitempty"` // for attribution
}
```

### 2.4 New API Handlers
- `GET /api/artistart/{encoded-artist}/candidates` - Search web sources for candidates (management)
- `POST /api/artistart/apply` - Download and save selected image to artist folder (management)
- `GET /api/artistart/{artist}/Artist.jpg` - Serve cached images (display)
- Static file serving: `/folder/{encoded-path}/Artist.jpg`

### 2.5 Artist Image Manager
Create `backend/internal/artistimage/manager.go`:
- `GetOrFetchArtistImage(artistName) (imagePath, error)` - Auto-fetch on first display
- `DownloadArtistImage(url, artistName) error` - Save to coverArtRoot
- `GetArtistImagePath(artistName) string` - Get filesystem path
- `ServeArtistImage(w, artistName)` - Serve via API

## 4. Implementation Steps

### Phase 1: Backend Core (Priority: HIGH)
1. Add `ArtistImageCandidate` model to `models/models.go`
2. Create `backend/internal/artistimage/` package
3. Implement `DiscogsProvider.GetArtistImage()` - uses artist search + artist details
4. Implement `MusicBrainzProvider.GetArtistImage()` - via URL relations
5. Add API endpoints in `api/handlers.go`:
   - `GET /api/artistart/{artist}/candidates` - for management modal
   - `POST /api/artistart/apply` - save selection
6. Add static file serving for artist images

### Phase 2: Auto-Fetch & Storage
1. Create `artistimage/manager.go` with:
   - `GetOrFetch(artistName) (path string, error)` - auto-fetches on first access
   - Uses in-memory cache to avoid repeated fetches
2. Implement download and save to `{CoverArtRoot}/{artist}/Artist.jpg`
3. Handle existing file cleanup (artist.jpg, artist.jpeg, Artist.png)

### Phase 3: Frontend Display
1. Update `SearchView.vue` artist section to show image
2. Update `ArtistsView.vue` to show artist image
3. Add image URL construction utility

### Phase 4: Management UI (Artist Page)
1. Create artist detail page or extend existing
2. Add "Update" button
3. Create candidate selector modal (reuse CoverArtSelector pattern)
