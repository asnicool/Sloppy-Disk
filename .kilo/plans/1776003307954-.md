# Plan: Legacy Static HTML + Vanilla ES5 Music Player Page

## Project Context

This is a **Vue.js + Go (MPD client)** project. The user wants a NEW standalone legacy page with maximum iOS 10.3 compatibility.

## User Requirements (Confirmed)

- Vanilla ES5 JavaScript (no Alpine.js)
- Plain, simple HTML (no Astro)
- Serve as static files alongside existing frontend
- No modifications to Go backend

---

## Solution Overview

Create a standalone `legacy/` folder with plain HTML + ES5 JavaScript that calls the existing Go backend API endpoints.

### Directory Structure
```
frontend/public/legacy/
├── index.html       # Main page
├── style.css      # Basic CSS
└── app.js         # ES5-compatible JavaScript
```

### API Endpoints to Use

| Endpoint | Method | Purpose |
|----------|--------|---------|
| `/api/status` | GET | Get playback status |
| `/api/play` | POST | Play |
| `/api/pause` | POST | Pause |
| `/api/stop` | POST | Stop |
| `/api/next` | POST | Next track |
| `/api/previous` | POST | Previous track |
| `/api/albums/random` | GET | Get 10 random albums |
| `/api/search` | GET | Search database |
| `/api/playlist` | GET | Get queue |

---

## Implementation Plan

### 1. index.html
- Simple HTML5 with semantic structure
- Sections: Header (status), Search form, Random albums grid, Queue display
- Inline or linked basic CSS
- Script tag at end of body

### 2. app.js (ES5-compatible)
- Use `XMLHttpRequest` for API calls (no fetch)
- Use `setInterval` polling every 2-3 seconds for status updates
- Manual DOM manipulation via `getElementById` / `querySelector`
- Event handlers with plain functions (no arrow functions)

### 3. style.css
- Basic responsive grid for albums
- Simple player controls bar
- Search input and results styling
- CSS2.1 compatible (avoid flexbox if needed for max compatibility)

### Features to Implement

1. **Play Status Display**
   - Show: state (playing/paused/stop), current song, elapsed time
   - Poll `/api/status` every 2-3 seconds

2. **Control Buttons**
   - Play/Pause toggle
   - Previous/Next buttons
   - POST to respective endpoints

3. **Database Search**
   - Input field for search query
   - Call `/api/search?q=<query>`
   - Display results with "Enqueue" button per result

4. **10 Random Albums**
   - Call `/api/albums/random` on page load
   - Display as grid with album art, title, artist
   - "Play" and "Enqueue" buttons per album

---

## iOS 10.3 Compatibility Checklist

- [ ] No ES6 features (const, let, arrow functions, template literals)
- [ ] Use `var` only
- [ ] Use `function() {}` for functions
- [ ] Use `XMLHttpRequest` instead of fetch
- [ ] Use callbacks instead of Promises/async-await
- [ ] Use `setInterval` instead of WebSocket
- [ ] Use string concatenation instead of template literals
- [ ] Basic CSS (simple selectors, no modern properties)

---

## Deployment

Static files go in `frontend/public/legacy/` - served automatically by Go backend at `/legacy/` route (or configure as needed).

---

## Execution Steps

1. Create `frontend/public/legacy/index.html`
2. Create `frontend/public/legacy/style.css` 
3. Create `frontend/public/legacy/app.js` with ES5 JavaScript
4. Test by loading page and verifying API calls work

No backend changes required.
